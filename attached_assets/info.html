Introduction

The StreamX File Download API is a RESTful web service that allows you to search for files on the scloud.ninja platform and retrieve direct download links for those files. This API is designed to be simple and easy to use.

The API utilizes standard HTTP methods and returns responses in JSON format. All endpoints are publicly accessible and do not require authentication.

API Endpoints

The API provides the following endpoints:

GET /api/search

Search for files on scloud.ninja using a query string.

GET /api/download

Retrieve detailed information and a download link for a specific file using its ID.

GET /health

Health check endpoint to verify that the API is running.

Search API

The search API allows you to find files based on a search query.

Request

GET /api/search?search=your_search_query
Parameters
Parameter Type Required Description
search string Yes The search term to find files (e.g., movie titles, file names)
Response

Returns an array of file objects with the following properties:

Property Type Description
name string The name of the file
size string The size of the file (e.g., "1.2 GB")
link string The file ID to be used with the download API
Example Response
[
  {
    "name": "Example Movie (2023).mkv",
    "size": "2.5 GB",
    "link": "abc123"
  },
  {
    "name": "Another File.mp4",
    "size": "1.8 GB",
    "link": "def456"
  }
]

Download API

The download API retrieves detailed information about a specific file and provides a direct download link.

Request

GET /api/download?id=file_id
Parameters
Parameter Type Required Description
id string Yes The file ID obtained from the search API
Response

Returns a file object with the following properties:

Property Type Description
filename string The full name of the file
size string The size of the file (e.g., "1.2 GB")
downloadLink string The direct URL to download the file
Example Response
{
  "filename": "Example Movie (2023) 1080p.mkv",
  "size": "2.5 GB",
  "downloadLink": "https://example.com/download/abc123"
}

Usage Examples

cURL Examples

Search for Files
curl -X GET "https://your-api-url.com/api/search?search=avengers"
Get Download Link
curl -X GET "https://your-api-url.com/api/download?id=abc123"

JavaScript Examples

Search for Files
// Search for files
fetch('/api/search?search=avengers')
  .then(response => response.json())
  .then(data => {
    console.log('Search results:', data);
  })
  .catch(error => {
    console.error('Error:', error);
  });
Get Download Link
// Get download link for a specific file
fetch('/api/download?id=abc123')
  .then(response => response.json())
  .then(data => {
    console.log('File details:', data);
    window.open(data.downloadLink, '_blank');
  })
  .catch(error => {
    console.error('Error:', error);
  });

Error Handling

The API uses standard HTTP status codes to indicate success or failure of a request. In case of an error, the response will contain a JSON object with an error message.

Example Error Response
{
  "error": "Failed to fetch search results: Invalid request"
}
Common Error Codes
Status Code Description
400 Bad Request - Missing required parameters
404 Not Found - The requested resource was not found
500 Internal Server Error - Something went wrong on the server
